# Agent Chat Analyze 要件定義書

## 1. 概要

本システムは、AIエージェント（特にClaude Code）との対話内容を分析し、ユーザーフィードバックの傾向や作業の進展/阻害要因を識別するツールである。

## 2. EARS形式による要件定義

### 2.1 機能要件

#### FR-001: 対話ログ解析機能
**WHEN** ユーザーがClaude Code形式の対話ログファイルを指定した場合、
**THE SYSTEM SHALL** 対話ログを解析してDuckDBに構造化データとして格納する。

#### FR-002: フィードバック傾向分析機能
**WHEN** ユーザーがフィードバック分析を実行した場合、
**THE SYSTEM SHALL** ユーザーのフィードバックパターンを分析し、傾向を識別する。

#### FR-003: 作業進展要因分析機能
**WHEN** ユーザーが作業分析を実行した場合、
**THE SYSTEM SHALL** 作業を進展させた行動と阻害した行動を分類・分析する。

#### FR-004: プロンプト生成機能
**WHEN** 分析結果が得られた場合、
**THE SYSTEM SHALL** 改善されたプロンプトを自動生成する。

#### FR-005: レポート出力機能
**WHEN** 分析が完了した場合、
**THE SYSTEM SHALL** 結果をマークダウン形式のレポートとして出力する。

#### FR-006: コマンドライン操作機能
**WHEN** ユーザーがコマンドラインでツールを実行した場合、
**THE SYSTEM SHALL** 対話的なインターフェースを提供する。

### 2.2 非機能要件

#### NFR-001: パフォーマンス要件
**THE SYSTEM SHALL** 1000件の対話ログを10秒以内に処理する。

#### NFR-002: 拡張性要件
**THE SYSTEM SHALL** 将来的に他のAIエージェント形式（ChatGPT等）にも対応可能な設計とする。

#### NFR-003: 保守性要件
**THE SYSTEM SHALL** テストカバレッジ80%以上を維持する。

#### NFR-004: 品質要件
**THE SYSTEM SHALL** 型安全性を保証するため、すべての関数に型ヒントを提供する。

## 3. システム境界

### 3.1 対象範囲（In Scope）
- Claude Code形式の対話ログ解析
- フィードバック傾向分析
- 作業進展/阻害要因分析
- プロンプト改善提案
- マークダウンレポート生成
- コマンドラインインターフェース

### 3.2 対象外（Out of Scope）
- リアルタイム対話ログ監視
- WebUI提供
- 他形式（ChatGPT、Gemini等）の対話ログ解析（初期バージョン）
- 対話ログの自動収集機能

## 4. 技術要件

### 4.1 開発環境
- Python 3.9以上
- DuckDB（分析用データベース）
- Click（CLI作成）
- Pydantic（データモデル）
- pytest（テストフレームワーク）

### 4.2 出力形式
- マークダウン形式のレポート
- コンソール出力（進捗表示）

## 5. 制約事項

### 5.1 技術的制約
- Python 3.9以上の環境が必要
- DuckDBがサポートする環境での動作

### 5.2 運用制約
- 対話ログファイルは手動で提供する必要がある
- 分析結果の解釈はユーザーが行う

## 6. 受入基準

### 6.1 基本機能
- [ ] Claude Code形式の対話ログを正常に解析できる
- [ ] フィードバック傾向を適切に分析できる
- [ ] 作業進展/阻害要因を識別できる
- [ ] 改善プロンプトを生成できる
- [ ] マークダウンレポートを出力できる

### 6.2 品質基準
- [ ] 全テストが通過する
- [ ] テストカバレッジが80%以上
- [ ] 型チェックエラーがない
- [ ] コードスタイルが統一されている

### 6.3 運用基準
- [ ] コマンドラインから実行できる
- [ ] エラーメッセージが分かりやすい
- [ ] ヘルプ機能が提供されている
