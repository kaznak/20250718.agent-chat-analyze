# Agent Chat Analyze 要件定義書

## 1. 概要

本システムは、AIエージェント（特にClaude Code）との対話内容を分析し、ユーザーフィードバックの傾向や作業の進展/阻害要因を識別するツールである。

## 2. EARS形式による要件定義

### 2.1 機能要件

#### FR-001: 対話ログ解析機能
**WHEN** ユーザーがClaude Code JSONL形式の対話ログファイル（`~/.claude/projects/*/session-id.jsonl`）を指定した場合、
**THE SYSTEM SHALL** 対話ログを解析してDuckDBに構造化データとして格納する。

#### FR-002: フィードバック傾向分析機能
**WHEN** ユーザーがフィードバック分析を実行した場合、
**THE SYSTEM SHALL** ベクトル埋め込みを使用してユーザーのフィードバックパターンを分析し、意味的類似性に基づいて傾向を識別する。

#### FR-003: 作業進展要因分析機能
**WHEN** ユーザーが作業分析を実行した場合、
**THE SYSTEM SHALL** 作業を進展させた行動と阻害した行動を分類・分析する。

#### FR-004: プロンプト生成機能
**WHEN** 分析結果が得られた場合、
**THE SYSTEM SHALL** 改善されたプロンプトを自動生成する。

#### FR-005: レポート出力機能
**WHEN** 分析が完了した場合、
**THE SYSTEM SHALL** 結果をマークダウン形式のレポートとして出力する。

#### FR-006: ベクトル埋め込み生成機能
**WHEN** 対話ログが解析される際、
**THE SYSTEM SHALL** 各メッセージに対してベクトル埋め込みを生成し、DuckDBのベクトル型フィールドに格納する。

#### FR-007: 意味的類似性検索機能
**WHEN** ユーザーが類似フィードバックを検索する場合、
**THE SYSTEM SHALL** コサイン類似度を使用してベクトル空間で意味的に類似したメッセージを検索する。

#### FR-008: コマンドライン操作機能
**WHEN** ユーザーがコマンドラインでツールを実行した場合、
**THE SYSTEM SHALL** 対話的なインターフェースを提供する。

### 2.2 非機能要件

#### NFR-001: パフォーマンス要件
**THE SYSTEM SHALL** 1000件の対話ログを10秒以内に処理する。

#### NFR-002: 拡張性要件
**THE SYSTEM SHALL** 将来的に他のAIエージェント形式（ChatGPT等）にも対応可能な設計とする。

#### NFR-003: 保守性要件
**THE SYSTEM SHALL** テストカバレッジ80%以上を維持する。

#### NFR-004: 品質要件
**THE SYSTEM SHALL** 型安全性を保証するため、すべての関数に型ヒントを提供する。

#### NFR-005: ベクトル処理要件
**THE SYSTEM SHALL** 384次元のベクトル埋め込みを使用し、コサイン類似度計算を5秒以内に完了する。

#### NFR-006: 埋め込みモデル要件
**THE SYSTEM SHALL** 日本語と英語の両方に対応した多言語埋め込みモデルを使用する。

## 3. システム境界

### 3.1 対象範囲（In Scope）
- Claude Code JSONL形式の対話ログ解析（`~/.claude/projects/*/session-id.jsonl`）
- sessionIdベースの対話グループ化
- ベクトル埋め込みを使用した意味的分析
- コサイン類似度による類似フィードバック検索
- 多言語対応（日本語・英語）のテキスト埋め込み
- フィードバック傾向分析
- 作業進展/阻害要因分析
- プロンプト改善提案
- マークダウンレポート生成
- コマンドラインインターフェース

### 3.2 対象外（Out of Scope）
- リアルタイム対話ログ監視
- WebUI提供
- 他形式（ChatGPT、Gemini等）の対話ログ解析（初期バージョン）
- 対話ログの自動収集機能

## 4. 技術要件

### 4.1 開発環境
- Python 3.9以上
- DuckDB（分析用データベース、ベクトル機能使用）
- Click（CLI作成）
- Pydantic（データモデル）
- pytest（テストフレームワーク）
- sentence-transformers（テキスト埋め込み生成）
- transformers（事前学習済みモデル）

### 4.2 Claude Code JSONL形式の詳細
- **保存場所**: `~/.claude/projects/{project-name}/{session-id}.jsonl`
- **形式**: JSONL（JSON Lines）
- **主要フィールド**:
  - `sessionId`: 対話セッションの識別子
  - `type`: メッセージタイプ（"user" | "assistant"）
  - `message`: メッセージ内容とロール情報
  - `timestamp`: ISO8601形式のタイムスタンプ
  - `uuid`: メッセージの一意識別子
  - `parentUuid`: 親メッセージの識別子
  - `cwd`: 作業ディレクトリ
  - `version`: Claude Codeのバージョン

### 4.3 ベクトル埋め込み仕様
- **モデル**: sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
- **次元数**: 384次元
- **対応言語**: 日本語、英語（多言語）
- **類似度計算**: コサイン類似度
- **データベース**: DuckDB FLOAT[384]型
- **インデックス**: HNSWベクトルインデックス（高速検索）

### 4.4 出力形式
- マークダウン形式のレポート
- コンソール出力（進捗表示）

## 5. 制約事項

### 5.1 技術的制約
- Python 3.9以上の環境が必要
- DuckDBがサポートする環境での動作

### 5.2 運用制約
- 対話ログファイルは手動で提供する必要がある
- 分析結果の解釈はユーザーが行う
