# 一般的な注意事項

- 日本語で話すこと
- t-wada の推奨する TDD のアプローチに基づいて進めること
- テストを実装したら github actions でテストが実行されるようにすること
- こまめにコミットしながら進めること
- 適当なタイミングで push し、PR を作成すること
- push したらトリガされるワークフローがある場合は監視し、問題があれば対応すること
- プルリクエストのマージの際は merge commit しブランチを削除すること
- 開発ステップごとにブランチを作成して作業すること
- ブランチを作成する前にベースブランチを確認すること
- EARS で要件を記述すること
- github のリリースページに表示される説明は必ず記述すること

- python は venv による仮想環境を構築して使用する事
